荣格性格哲学-心理类型

发表于 2021-01-02 更新于 2021-01-04 分类于 心理学

两种基本类型

心理学的一个重要问题是人格能不能被拆分成几个基本类型，而一个人可以的性格要素或许像三元素构成千百种颜色一样尤这几种基本结构调配而成？
这个解决方案存在于科学之前，比如星相图。荣格对它再分类的途径是否比星相图更为合理？这个问题就如同问以弦理论描述宇宙是否比无数大乌龟一个个驮着宇宙更合理一样。

荣格分成两类基本类型：

  1. 外向型：不断以各种方式拓展自己
  2. 内向型：抵御外界要求的倾向

要理解这两种类型依据字面解释已经足够，但是这两种类型如同轨道，可能是汽车轨道，也可能是地铁，所以定义性格的交通方式还依赖其心灵的功能和它做笛卡尔乘积。

心灵功能

感觉

感官知觉，通过体感传达的知觉。这种感觉可一分为二

  1. 具体感觉：与表象，情感，思维混合，作为整体出现。如看见花时喜恶的自我感受，关于根茎叶浮现的意象混合而成的是具体感受
  2. 抽象感觉：从对象差异分离出来，达到纯粹化。花的气味/形状/颜色成为花的特征，这就是抽象感觉。

感觉被荣格视为有意识的知觉：因为涉及意识的运用，在外向性格中感觉受到抑制，因为所看所感如果不能达到意识感受的阈值，它将不能从体感形成感觉，如“江上之清风，山间之明月，耳得之而为声，目遇之而成色”，这其中所谓的声和色并非荣格所言的感觉，因为他们并没有在意识中留下痕迹；在内向性格中感觉依然有抑制，这是因为若要形成感觉，内向型的人需要调用主观感受，这意味着不同人对于同一事物的感觉发展不同，就像不同的火候与不同的配料对于同样的食材产生了不一样的化学作用。

直觉

直觉的确定性依赖于一种明确的心理事实，只不过主体对这一事实的根源及其完备状态没有完全意识到而已，直觉大概就像我们对于欧几里得提出的五大公理的承认，对于牛顿三定律和万有引力存在的承认，这些不涉及逻辑的推导，但也不是对于苹果向下坠落这一感觉的再描述，而是存在于无意识的知觉。和感觉相同，荣格将直觉与外向内向去探索直觉的不同发展：

  1. 外向直觉指向世界：直觉瞬间遍历最多可遍历的可能性，排除所有感觉因素带来的偏差，直觉就像围棋里的妙手，直达客观事物的中真正潜伏的东西
  2. 内向直觉指向内心：人类内心积淀了人类社会万年的经验，心灵原型又由于我们的理解方式无法看清，但直觉却构成了神迹的可能性。那些看似预言式的的道理，正是内向直觉和心灵原型联系的作证和显现。

思维

思维是一种根据自身的规律用某种方式表达概念关系的心里功能，是一种统觉活动，并由此区分主动和被动的思维活动。如何判断一种功能是思维的产物，只需要看连接这一功能实现的是否是纯粹的概念性的东西。

  1. 外向思维：当思维有客观倾向即是外向的，这种思维考虑客观事物的联系和实现，不恰当地类比为工程家思维，即着眼于现有事物的实现，但发展到极致后会令人觉得缺乏自由和远见，关注在思维的具体一层却忽视了其中更深刻的链条和纵深。
  2. 内向思维：当思维有主观倾向即是内向的，这种思维不关注感觉和真正存在的客体而将所有自我心灵看到的意象分支抽象出来。我把它类比为数学家思维，将所有的抽象用心语和符号表示，再此基础上再次抽象，到了某种偏颇境地就会彻底脱离现实而找不到和现实相关的着力点。

如果内向思维发展到脱节，就会如《罪与罚》里的拉斯柯尔尼科夫一般，在自我内心中成就了尼采式的超人逻辑，背弃了一切时间法律秩序的意义。这看上去或许是一种深思，但其实已经走入了无可挽回的虚无。

情感

情感是发生在自我与某一特定内容之间的一种过程，是在接受或拒绝意义上给某一内容以确定价值的过程。情感是理性而不是非理性的，这是人类在自我价值认可过程中最核心的一种功能。完全抽象的情感全人类是共通的，这种情感这也是人性和兽性的分野处。

  1. 外向情感：受制于客观情况。情感确定的是自我价值，客观情况可以提供这种印证也可以剥夺自我部分，而当自我的情感完全依赖于客观情况时，便陷入了自我的完全脱离，所有的冷漠合理化，丧失了人类原型就无法去谈道德，谈愿望，丧失自我的人便陷入了歇斯底里症。贪痴嗔都是如此，被金钱，感情，不义所迷惑，完全投身于他们时自我的概念就不复存在，只剩下一个空洞的人格。
  2. 内向情感：依赖主观因素。这种感情的强度只能被神化——却永远被清楚理解，使人们沉默，难以接近。提出否定的情感判断，或以故作高深的冷漠作为防御。自恋症的出发都是由于这种内在情感：人类对于认同的需要是无穷的，所以权力欲也是无穷的，个人在情感失衡后便会完全倾向无限的自我牺牲，同时伴随权力的膨胀，便诞生了希特勒，斯大林这样的人。

八种功能性人格划分

人格类型 突出表现 动机 典型
外向感觉型 具体对象实际经验的积累 享乐和刺激感觉（可能处于个人道德动机） 宗教狂热分子
外向直觉型 穷尽所有人生可能性 不断浮现的可能性的强迫性动机 冒险家
外向思维型 关注客观事实 具体化自己的思维 科学研究者
外向情感型 产生的情感与普遍价值相符 对美的追求 欣赏古典乐
内向感觉型 关注事物效果和内心感受 外部世界无味，内心世界丰富 艺术家
内向直觉型 重视内在意象 潜意识心理的遗传基础 先知
内向思维型 不关注事实，而关注观点 形成光辉理念，达到真实的愿望 哲学家
内向情感型 关注与自我相关的东西 价值取向以自我为中心 思想家

关于外向思维是否能完全的解释世界，有一段这样的论述摘录如下：
外向思维由于过度的客观规定而从属于客观与料时，它就完全迷失在个别经验值中，累积的未经消化的经验与料。造成的思维分离通常需要心理补偿，把堆积起来但缺乏内在联系的整体连贯起来，诸如‘能量’等概念。对这种贫乏思维的补偿是令人印象深刻的事实积累，这些事实环绕一个狭限而频繁的观点，许多事物的有价值的面向被完全忽略了

划分人格不意味着好坏的划分，没有高低品级，只是注意到自己的人格类型便于更好的把握平衡，而达到个人性格的完善。

荣格性格哲学-自我探索

发表于 2021-01-02 更新于 2021-01-03 分类于 心理学

在人的整个一生中，他所应该做的，只是在这种固有人格基础上，去最大限度地发展它的多样性、连贯性和和谐性，小心谨慎着不让它破裂为彼此分散的，各行其是的相互冲突的系统。

关于人格如何形成，个性化如何塑造，个人精神问题如何治疗这些问题，几乎所有的心理学家都也没有确定的答案。这可能是个二难的问题，因为心理学面向的个性化必然使它的体系化和定理化存在困难，但弗洛伊德还是通过对梦的释义和分析完成了使心理学变为科学的第一步，尽管百年后再看荣格和弗洛伊德，或许研究方法不尽科学，但对于心理学是否使可以变成像物理和化学这样层层构建，环环相扣的问题，恐怕也没人能给出确切的答案。

心灵结构

心灵的每一次外在显现都同时混合着多种影响。首先是清醒的自我（the conscious ego）；其次是个人以及所从属的群体身上那些很少被意识到的情结；再次是自未被意识到的集体心理的那些无论以什么方式结合在一起的原动力机制

个人精神（spirit）：包括一个人所有的思想、感情和行为，无论它是意识的还是无意识的。精神即人生存在的土壤。

关于荣格对于人的人知我们可以看见东方哲学对它的影响，在现代科学将一切人的灵魂和精神结构到化学信号作用和电流刺激的时候，荣格则依然保持整体性的观念，即人的灵魂和精神是固有的，缺失了意识部分，无意识部分也会难以存在，因为精神不再存在。

精神的有深及浅的三个层次：

  1. 集体无意识
  2. 个人无意识
  3. 意识

无意识

精神活动本身都是无意识的，那些有意识的过程只不过事整个精神生活片段的片段和局部

从最基本的地方，荣格和弗洛伊德就出现了分歧，关于无意识的来源：

  • 弗洛伊德：无意识是被压抑被遗忘的心理内容的集合场所
  • 荣格：无意识具有非个人和超个人的特性

个人无意识

自我：自我是意识的门卫，如同一个过滤器，如果某种观念情感不被自我承认，那它就进入不了意识。

也正是因为自我的存在，昨天的我和今天的我才被我感觉为同一个人

个人无意识：自我的下面一层，容纳着所有与意识功能和自觉个性化不协调，不一致的心理活动和过程。

情结：个人无意识的内在构成，表示某种未被认同的对抗性的东西，既是障碍也是诱因。

个人无意识被情结所填充，它算是自我的源头，但其中也包含了自我不愿意接触的地方。而个人无意识中既包含了引诱，也包含了痛苦，它既是人生的动力来源，也是人生的焦躁来源。

集体无意识

整个人类都有对于救星的期待

集体无意识：人类经验的贮存所，又是这一经验的先天条件，乃是万古世象的一个意像。

集体无意识的三个基础:

  • 本能：表现为某种行为的自然冲动
  • 原型：一切心理反应的普遍形式，包括一切文化和科学领域。先验表达的可能性。领悟的典型模式。
  • 原始意象：于表象和原型之间，起凝聚和构型的作用。

关于这三者如何界定， 或许荣格也只是从例子中凭借直觉上对于过程的划分分成了三个部分:

  1. 在遇到一只蛇的时候，逃跑的冲动是人的本能；
  2. 但遇到一只无害的鸡的时候，出于某种幻想而害怕，这是原始意象；
  3. 而这种幻想是如何构建的，这必然存在一个内在的心灵机构，荣格归为了原型。

这种划分未经过实验验证，或许不是科学的，但是是符合直觉的。

Anima：男人的女性气质，诸如暧昧的情感和情绪，语言性的征兆，对非理性的接纳，容忍私人的爱意，对自然的感情。

  1. Anima过分消极：暴躁，意志消沉
  2. Anima过分积极：无丈夫气概
  3. Anima作用：调和内在价值

Animus：女人的男性气质，以兽性情感强迫别人，无情的力量

  1. Animus残忍鲁莽，顽固邪恶
  2. Animus富有创造力，通往自我价值，赋予精神稳定，和女人的心灵连结，更加创新

象征

超越和整合作用，使互相对立的相互冲突的情感处于有机统一的状态。消除内心分裂造成的焦虑，使人超越平庸肤浅的生活，帮助组织调动心理能量。

人类需要象征，象征是人类精神外化至物又再次反馈到内心的整个过程的综合，放在生活中说是我们的仪式感，而人类最重要的情感都在和象征挂钩。

  • 宗教情感：人们对于奇迹的相信（基督死而复生），和王子的一个吻就带回了白雪公主。这两个故事都象征了再生这一永恒的主题，它解决的的是人必然要死和人为何要生这一无解的冲突，放在死而复生这一循环之中，人得到了自我心灵的安慰甚至救赎，这是必然的。
  • 成年礼：人的个性化诞生基于人的自我意识浮现，但人的出生之后即在照料之中，它在人生的关键节点面对的是父母关系的淡化和父母的再定义，是过去自我与现在自我的分裂，面对的是生活和自我边界的再划分。而成年礼的形式大多是让人体会到强烈的死亡情绪，这是对过往自我的牺牲，也是对未来自我的重生。这一过程几乎出现在所有文化之中，不是一种巧合，而是人类心灵集体无意识部分共同性所导致的必然趋势。
  • 婚礼：婚礼并不是一种社会中彰显关系独占性的契约需求，也是人的心理需求。对于双方而言，都是放弃了个人自我意识部分强烈的自主权，而部分转向一种新形式的生命状态。

残酷而仁慈的双重原则：人对于危险的事物会天生趋近，人对于困难事物的解决会获得更多快乐。这种双重原则并不是风险利益的正相关描述，而是关于人类心灵古老预言的印证。西方最浪漫的神话“美女与野兽”中，贝尔的心灵觉醒正是双重原则的体现。贝尔问父亲要了一只白玫瑰，而白玫瑰却将父亲推向了危险的边缘，这是原则的第一次体现，最美丽的善良种在最危险的花园；贝尔在野兽将死之际许下了诺言，爱情中的温情和对动物人的恐惧构成了第二次原则体现。很明显原则是客观规律，难以改变，而人生得以存在正是存在于一传恐惧和期待并存的心灵觉醒之中。

整体性：我们很难找到理性的公式表达统一的理性和非理性，经验和超验的自我本质，只有象征可以体现这一整体性：

在原型的两极性中，意识和无意识，高尚和卑下，男性因素和女性因素相结合。他们通过自性象征表达着理性和非理性，善于恶，光明与黑暗…这正反应了难以捉摸的自我本质的对立统一特征

祸福相依

发表于 2020-12-31 更新于 2021-01-01 分类于 我

不敢忘2020

人生中需要这样一年，它来的时候夹杂些不和谐，去的时候也没涤荡清它来时带给我们的痛苦，这是种试炼也是个契机。就像那些过去的时间不会过去，它留下集体记忆的印记，也留下个人心路的一撇。

且做大概的总结，算是为自己留下一点回想，也是为新年准备一点思考。

时间段

  1. 1月~2月：在上海最冷的时候，在宿舍体会到了宋濂写的“手指不可屈伸”，而每天实习过后，读书准备毕业论文，又体会到了“弗之殆”。或许是个人需要一点戏剧感，每天读书的时候感觉就像在电影中，但没想到这是大学生活的句号。
  2. 2月~6月：疫情上半场，疫情的开始似乎与大部分人无关，但迅速就蔓延到每个人内心的黑暗之地，吞没着大众的理想与愿望。其中有英雄主义，有自私自利的人性，也有合理性与人性的斗争。疫情里人类只会失败，但人类不能停止斗争。我对大部分事无能为力，无风无浪地完成了毕业论文，拿到了不那么心动的offer，我大概就如2020，没那么好，但也没想的那么差。
  3. 7月~9月：2020教会了我去爱具体的人，而不是抽象的人。乌鲁木齐的疫情对它的民众而言是场战斗，更是场流放。如今无意指责，留下些许对时光的诚挚和对旧友的留恋，感恩些亲人的支持和新友的陪伴。我可能学会了诚挚。
  4. 10月~12月：虽然孤独陪伴着我，但音乐给了些许慰藉，作曲，弹琴是我人生中幸福的享受，对世界别那么对抗，多一些和解可能更好。只剩日常上课，读书，弹琴，锻炼，刷题，少了那些琐事，我用规律塑造了一个可能略显脆弱的自我，期望交流又沉溺孤独，渴望改变又更加专注。
  5. 2020对我而言产生的人格塑造，可能超过了大学前三年的改变。未来我希望能朝向勇敢的方向再走一点，少一点骄傲，多一点谦逊。

2020年，有些书重读，有些书精读，有些书读到最后近乎草草翻过。因为博客上线，明年对于精读的书会有笔记与个人感悟的分享，或许这样督促更加合理？

堆栈-最长有效括号

发表于 2020-12-31 更新于 2021-01-01 分类于 Algorithm and DataStructure

问题

给定一个只包含 ‘(‘ 和 ‘)’ 的字符串，找出最长的包含有效括号的子串的长度。

示例

输入: "(()"
输出: 2
解释: 最长有效括号子串为 "()"

输入: ")()())"
输出: 4
解释: 最长有效括号子串为 "()()"

分析

堆栈的解决思路：

面对括号的问题，第一感觉是类似clisp的处理：

准备一个stack，每进来一个左括号，就进行一次压栈，遇到右括号则进行一次退栈。

按照上述处理方法，每次退栈则意味着一次有效匹配，可是如何累积有效匹配使它可以面对”(())”[嵌套]和”()()”[连接]这两种模式？

  1. 首先确定我们把什么东西压栈，如果我们放进去的是一个个”(“字符，那完全可以准备变量而不是一个栈，所以我们先尝试放进去一个个左括号的index。
  2. 面对嵌套情况：我们用当前的i-stack.top+1表示长度，当最后一个’)’匹配了它对应的’(‘时我们可以更新maxLength。
  3. 面对连接情况：连接意味着我们需要记忆第一个匹配完成后的长度，在第二次匹配完成时我们可以把这两个结果串联成一个新的最大值。很明显记录单凭此次匹配的index并不能得到我们全部的信息。
  4. 假设当前pop的元素为a，考虑连接情况，我们知道(i-a+1)并不足以使我们得到结果，所以我们考虑使用pop后的stack.top
    • 嵌套情况：”((())”，最后一次匹配可以得到最长长度4-0=4
    • 连接情况：”(()()”，最后一次匹配可以得到最长长度4-0=4
  5. 综上我们用pop元素下面的那个元素作为计算匹配长度的辅助值，统一了两种情况。

双向遍历：

  1. 当从左到右遍历时，我们可以准备一个变量，一个’(‘使它加一，一个’)’使它减一（最小为0），那毫无疑问，当该变量为0时，说明这一串字符是合适的括号。
  2. 这还不够，它在一种情形下会失效，加入一段连续左括号的开始位置是k，到末尾的时候还不能使变量为0，那就没有匹配成功的括号，比如’((()’。
  3. 解决方式就是从右向左再遍历，出于对称性，左到右的遍历解决了右到左的最后一段错误，右到左亦然。

代码实现

1. 栈的方式

public int longestValidParentheses(String s) {
    Deque<Integer> stack = new LinkedList<>();
    int res = 0;
    int lstMatch = -1;
    for(int i=0; i<s.length(); i++){
        if(s.charAt(i) == '('){
            stack.push(i);
        } else {
            if(stack.isEmpty()){
                lstMatch = i;
            } else {
                stack.pop();
                if(stack.isEmpty()){
                    res = Math.max(res, i - lstMatch);
                } else {
                    res = Math.max(res, i - stack.peek());
                }
            }
        }
    }
    return res;
}

时间复杂度：$O(n)$

空间复杂度：$O(n)$

2. 双向遍历

    public int longestValidParentheses(String s) {
    int leftPar = 0;
    int lstMatch = 0;
    int curMatch = 0;
    int maxMatch = 0;
    int preMaxMatch = 0;
    for(int i=0; i<s.length(); i++){
        if(s.charAt(i) == '('){
            leftPar += 1;
        } else {
            if(leftPar == 0){
                curMatch = 0;
                lstMatch = 0;
            } else {
                leftPar -= 1;
                curMatch += 2;
                if(leftPar == 0){
                    lstMatch += curMatch;
                    curMatch = 0;
                    if(lstMatch > maxMatch)
                        maxMatch = lstMatch;
                }
            }
        }
    }
    preMaxMatch = maxMatch;
    lstMatch = 0;
    curMatch = 0;
    maxMatch = 0;
    leftPar = 0;
    for(int i=s.length() - 1; i>=0; i--){
        if(s.charAt(i) == ')'){
            leftPar += 1;
        } else {
            if(leftPar == 0){
                curMatch = 0;
                lstMatch = 0;
            } else {
                leftPar -= 1;
                curMatch += 2;
                if(leftPar == 0){
                    lstMatch += curMatch;
                    curMatch = 0;
                    if(lstMatch > maxMatch)
                        maxMatch = lstMatch;
                }
            }
        }
    }
    return Math.max(preMaxMatch, maxMatch);
}

时间复杂度：$O(n)$

空间复杂度：$O(1)$

动态规划-编辑距离

发表于 2020-12-30 分类于 Algorithm and DataStructure

问题

给你两个单词 word1 和 word2，请你计算出将 word1 转换成 word2 所使用的最少操作数 。
你可以对一个单词进行如下三种操作：

  • 插入一个字符
  • 删除一个字符
  • 替换一个字符

分析

编辑距离是针对二个字符串（例如英文字）的差异程度的量化量测，量测方式是看至少需要多少次的处理才能将一个字符串变成另一个字符串。编辑距离可以用在自然语言处理中，例如拼写检查可以根据一个拼错的字和其他正确的字的编辑距离，判断哪一个（或哪几个）是比较可能的字。DNA也可以视为用A、C、G和T组成的字符串，因此编辑距离也用在生物信息学中，判断二个DNA的类似程度。Unix 下的 diff 及 patch 即是利用编辑距离来进行文本编辑对比的例子。

编辑距离在语音识别，现代编辑器，输入法甚至基因测序中都有作用。在这个问题里出现的是Levenshtein距离，即可以对字符串进行insert,delete,replace三种操作完成变化。

刚拿到问题时第一直觉是尽可能原位置匹配，但这必然是个极其复杂的搜索过程，所以问题并没有得到明晰反而更复杂了；第二个思路是是否可能在一个最优变化中是否可以缩小到仅有插入替换或者删除替换，但存在”aosososoe”，”osososea”这样的反例，所以也不可行。既然不能把问题的结构简化也不能设计出看似合理的搜索树，那就尝试思考是否可以从子问题构建原问题的解。第三方案似乎可行，但如何拆分？

Optimal Substructure Property

因为存在两个字符串，所以这不是一个单维度的问题。所以如果使用DP一定会先考虑如果我现在有(word1[0:m-1], word2[0:n-1])的最小距离，是否可以构成(word1[0:m], word2[0:n])的最小距离？如果我的最后一步是替换或者word1[m-1]==word2[n-1]那应该就是可行的。似乎整理出了一点思路，但这还远远够不上最优子结构，因为:

  1. 虽然存在变化，但可能最后一步不是替换
  2. 整个变换根本不存在替换
  3. 这个子问题的解并不是通往最优大方案的必经方案。

这三个问题是否可解？

问题2：
只要字符串不一致，就一定存在三种变换之一。所以在找子问题的时候我可以扩大我的子问题范围，让最后一步可能是insert，delete，replace三者之一。

  1. 如果通过(word1[0:m-1], word2[0:n-1])方案, 最后一步只能是replace或者不变
  2. 如果通过(word1[0:m], word2[0:n-1])方案, 最后一步只能是insert
  3. 如果通过(word1[0:m-1], word2[0:n])方案, 最后一步只能是delete

问题1：
一个最优解的变化过程可以写成二元序列的形式，[(i_1,ope1), (i_2, ope2)…(i_n, open)]，ope为三操作之一，i为具体元素。无论我最后一个操作需要什么，只要在序列中出现了这个操作，都可以把它放到最后而整个变化过程依然成立。这是因为对个元素我们最多只会操作一次，插入后替换删除都是会引发多余步骤，删除后再插入亦然，替换后删除亦然。

问题3:
现在所有的子问题齐全，我们也证明了确实存在从三个子最优方案到达原方案的路径。接下来要看是否存在一条路径，使得其不通过我们的三条最优子方案：

  1. 这条路径的最后操作如果是replace，则必然存在于(word1[0:m-1], word2[0:n-1])的方案中，
  2. 这条路径的最后操作如果是delete，则必然存在于(word1[0:m], word2[0:n-1])的方案中，
  3. 这条路径的最后操作如果是insert，则必然存在于(word1[0:m-1], word2[0:n])的方案中，

解决这三个问题也就证明了我们的算法是拥有最优子结构性质，也就是动态规划可解的。

转移方程:

实现

def minDistance(self, word1, word2):
    """
    :type word1: str
    :type word2: str
    :rtype: int
    """
    len1 = len(word1)
    len2 = len(word2)
    dp = [[0 for j in range(len1 + 1)] for i in range(len2 + 1)]
    for i in range(len1 + 1): 
        dp[0][i] = i
    for i in range(len2 + 1):
        dp[i][0] = i
    for i, ele1 in enumerate(word2):
        for j, ele2 in enumerate(word1):
            tmp = 1
            if ele1 == ele2:
                tmp = 0
            dp[i + 1][j + 1] = min(dp[i][j] + tmp, dp[i][j+1] + 1, dp[i+1][j] + 1)
    return dp[len2][len1]

动态规划理论

发表于 2020-12-30 分类于 Algorithm and DataStructure

Definition

Break up a problem into a series of overlapping subproblems, and build up solutions to larger and larger subproblems.

动态规划是一种经典的算法思想，似乎说到它就会浮现一系列问题，LCS，背包问题，但这都是算法层面的实现，涉及具体的方法。而本文不详细讲解具体问题，仅以逻辑和直觉上的正确性出发如何对这个算法思想和其它算法思想做出界定，并说明这个思想应用的问题范围。

之前遇到了这样一个判断，“分治能解决的问题动态规划都可以解决”，这个问题似乎易于回答，“因为我们不用动态规划去做merge sort，所以判断为假”。但这个判断核心的关键是在于对overlap的理解上，什么是拆分成一系列重叠的子问题？

Optimal Substructure Property

A problem exhibits optimal substructure if an optimal solution to the problem contains within it optimal solutions to subproblems.

这个问题的前提条件是面对对象是优化问题，动态规划解决的一定是最优化问题，而排序很明显，并不是一种最优化问题，这就是动态规划和分治最重要的区别，二者都依赖子问题与父问题结构的相似，但除了思想上这部分的重叠外，两者的解决问题对象全然不同。

大问题的解决方案蕴含在子问题方案之中，这就是动态规划的核心要求。换言之，如果我们构造出的子问题方案，基于该子问题方案找到最优大问题方案之中后，存在对于该子方案的另一选择可以构造出更优的全局结果，这个问题便不是DP适用的。还是以排序举例，如果将排序问题转换成求解最小的$\Sigma^n_{i=1}a_i*i$，似乎就将排序转换成了一个最优化问题，那我们上面的否定就站不住脚。但是，问题的解决方案是否包含在子问题的解决方案之中？对于数组A一分为二拆为B1,B2，那B1的结果和B2的结果的最优值B1min,B2min并没有结合方案去获得A的最优值。

Overlapping subProblem

DP的子问题一定是互相重叠的，这似乎有一点矛盾，既然我拆成子问题，那子问题必然不能在状态上互相影响，否则对于他们的求解就是更为复杂的，那如何才是重叠？重叠指示的并不是问题的互相耦合，而是不同大小问题所面对的子问题可能是相同的，比如在求解Fib(10)时拆分成了Fib(9)和Fib(8),而这两个子问题都需要求解Fib(7)，即二者重叠。更直接的说法，Subproblems are smaller versions of the original problem. 再以merge sort争辩举例，如果不以最优值结合，而以最优方案结合，将拆分出的B1数组和B2数组以merge算法结合到一起就是A的解，这不就是符合DP要求了吗？但这依然不符合DP的思想，因为无论如何拆分，B1和B2不会存在重叠，按照merge sort算法，不会记录一个序列既是B1的解又是B2的解，所以这便不是DP的。

Method

使用DP的核心思路有两个：

  1. Memorization:自顶向下方法使用的是记忆化，直观上而言，DP记忆化了搜索树上出现多次的子问题，所以算法在面对每个子问题的时候，先去考虑是否是见过的（重叠子问题）,如果是使用记忆化数组访问当时的解

     dp = []
 func Fib(int n):
     if(dp[n-1]) 
         ele1 = dp[n-1]
     else:
         ele1 = Fib(n-1)
     if(dp[n-2]) 
         ele2 = dp[n-2]
     else:
         ele2 = Fib(n-2)
     dp[n] = ele1 + ele2
     return dp[n]

  1. Tabulation：自下向上问题使用的是table，从最小的子问题开始，逐渐构建到更大的问题，最后达到当前问题规模则求解完毕。

     func Fib(int n):
     dp = [1, 1]
     if(n == 1) return 1
     if(n == 2) return 1
     for(i=2; i<n; i++)
         dp[i] = dp[i-1] + dp[i-2]
     return dp[n-1]

数学归纳法

DP的思路看上去和数学归纳法很近似，在很多时候，dp的问题以数学归纳法的证明方式写出可以帮助整理程序思路，并直觉上判断其是否是最优子结构。

数学归纳法证明步骤如下:

  1. 证明若n=1，2…k时存在最优方案
  2. 证明n=m+1假设存在最优方案，则n=m+1时最优方案可以从n=m最优方案中获得

论题：可用数学归纳法证明则可用最优子结构证明：

证明：

1. 令S为n=m+1时的问题，A为n=m时的问题
2. 如果我们找到一个S的解决方案不使用A的最优解决方案，那根据数归证明2，该方案最优性<=基于A方案获得的S方案

总结

DP需要大量的经验，其中转移方程的构建，dp思想和其它算法的结合，对于问题动规可能性的直觉，尽是垒土之功。
